Design decisions (5-heart cap):

• Exactly 5 visible hearts always — never more, never less (even at 0 lives, 5 empty hearts shown)

• Overflow text = maxLives - 5 (the hidden pool), colored per dungeon theme

• Ratio text (e.g. "10/15") always shown in small grey — true lives / true max

• Dim/grey 5th heart when lives ≥ maxLives - 3 — "close to full, use potions"

• Gold glow on all 5 hearts + gold overflow when at max — "fully healed, next heal = +50 score"

• No layout break — the HUD row stays the same width regardless of dungeon

• lives_boost artifact still works: adds +1 to the dungeon's base maxLives (affects overflow text)

• At 0 lives: 5 empty hearts + overflow text — still communicates the pool size even when dead
